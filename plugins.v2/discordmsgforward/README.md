# Discord消息转发

轮询 Discord 频道新消息，转发到 MoviePilot 的指定通知渠道（**可多选**：微信 / Telegram / Slack 等），支持每频道独立规则、关键词过滤、兑换码正则提取、图片转发、消息聚合与自定义消息模板。典型场景：把游戏官方 Discord 的礼包码公告（如寒霜启示录 WOS giftcode 频道）转发到微信。

## 功能

- **多频道监听**：每行一个频道，支持 `频道ID#备注#关键词#转发渠道` 扩展语法，单频道可覆盖全局规则
- **转发渠道多选**：渠道列表来自 设定→通知 中已启用的通知渠道，留空发送到全部渠道
- **图片转发**：消息附件图片 / Embed 配图作为通知图片一并推送
- **消息聚合**：一次轮询到多条新消息时合并为一条通知，避免刷屏
- **自定义模板**：标题/内容模板支持 `{channel}` `{author}` `{content}` `{codes}` `{time}` `{count}` 变量
- **兑换码提取**：可选正则，命中的兑换码在通知中单独列出并记录在历史页
- **远程命令**：交互渠道发送 `/discord_check` 手动触发检查
- 通过 Discord REST API 轮询（Bot Token），无需常驻 WebSocket；首次运行只记录基线，不刷历史消息；支持系统代理；详情页展示转发历史

## 准备工作

1. **创建 Bot**：打开 [Discord 开发者平台](https://discord.com/developers/applications) → New Application → 左侧 Bot：
   - `Reset Token` 获取 Bot Token（只显示一次，保存好）
   - 开启 **MESSAGE CONTENT INTENT**（否则拉不到消息正文）
2. **把 Bot 拉进自己的服务器**：左侧 OAuth2 → URL Generator → Scopes 勾 `bot` → Bot Permissions 勾 `View Channels`、`Read Message History` → 打开生成的 URL，选择自己的服务器授权。
3. **获取频道 ID**：Discord 客户端 设置 → 高级 → 开启「开发者模式」，右键目标频道 → 复制频道 ID。
4. 想监听**别人服务器**的频道（无法拉 Bot）：如果对方是公告频道（有 Follow/关注 按钮），先把它关注到自己服务器的某个频道，再监听自己这个频道即可。

## 配置说明

| 配置项 | 说明 |
|--------|------|
| Bot Token | Discord 开发者平台创建的 Bot Token |
| 监听频道 | 每行一个：`频道ID#备注#关键词#转发渠道`，后三段可省略；关键词/渠道多个用 `\|` 分隔，留空用全局配置 |
| 转发渠道 | 多选；留空 = 全部启用的通知渠道 |
| 通知类型 | 转发消息使用的 MP 通知类型，默认「插件」；所选渠道的消息类型开关需包含该类型 |
| 消息聚合 | 同一频道一次轮询的多条消息合并为一条通知 |
| 图片转发 | 附件图片 / Embed 配图作为通知图片推送 |
| 全局关键词过滤 | 逗号或 `\|` 分隔，留空转发全部；频道级关键词优先 |
| 兑换码提取正则 | 留空不提取，如 `[A-Za-z0-9]{6,20}` |
| 标题/内容模板 | 变量见上；兑换码为空时，含 `{codes}` 的行自动隐藏 |

**频道行示例：**

```
1234567890123456789#WOS礼包码
1234567890123456789#WOS礼包码#code|gift#微信
9876543210987654321###Telegram|Slack
```

## 常见问题

- **HTTP 401**：Token 错误或已重置。
- **HTTP 403**：Bot 不在该服务器，或没有「查看频道 / 阅读消息历史」权限。
- **消息正文是空的**：没开 MESSAGE CONTENT INTENT。
- **收不到通知**：确认 设定→通知 渠道正常，且所选「通知类型」在目标渠道的消息类型开关里已打开。
- **微信里图片不显示**：企业微信服务器需要能访问 Discord CDN 图片地址，个别情况下图裂属正常，正文不受影响。
