# Discord消息转发

将 Discord 频道新消息转发到 MoviePilot 的指定通知渠道（**可多选**：微信 / Telegram / Slack 等）。通用消息转发插件，支持频道下拉选择、关键词/作者/屏蔽词过滤、图片转发、消息聚合、免打扰时段、自定义模板与失败告警。

典型场景：游戏官方公告/礼包码频道转发到微信、社区重要通知汇总、只转发特定 Bot 的推送等。

## 三步上手

1. 填 **Bot Token**，保存 → 插件自动拉取 Bot 可见的频道列表
2. 重新打开配置页，**下拉勾选**要监听的频道
3. 选**转发渠道**（留空 = 发到全部启用的通知渠道），启用插件即可

## 功能

- **频道选择**：保存 Token 后自动拉取「服务器 / #频道」列表下拉多选；也可手动填 ID
- **高级规则**：手动行支持 `频道ID#备注#关键词#转发渠道`，单频道可覆盖全局规则（多值用 `|` 分隔）
- **转发渠道多选**：渠道列表来自 设定→通知 已启用的通知渠道
- **过滤规则**：关键词白名单、屏蔽词黑名单、作者白/黑名单（用户名精确匹配、不分大小写）
- **图片转发**：附件图片 / Embed 配图作为通知图片推送
- **消息聚合**：一次轮询多条新消息合并为一条通知
- **免打扰时段**：如 `23:00-08:00`，时段内消息暂存，结束后按频道汇总推送
- **跳转链接**：通知点开直达 Discord 原消息
- **失败告警**：连续 3 次轮询全部失败时推送提醒，恢复后自动重置
- **内容提取**：可选正则（如礼包码 `[A-Za-z0-9]{6,20}`），命中内容单独列出并记录历史
- **自定义模板**：标题/内容模板支持 `{channel}` `{author}` `{content}` `{codes}` `{time}` `{count}`
- **远程命令**：交互渠道发送 `/discord_check` 手动触发检查
- 首次运行只记录基线不刷历史消息；REST 轮询无需常驻连接；支持系统代理；详情页展示转发历史

## Bot 准备

1. [Discord 开发者平台](https://discord.com/developers/applications) → New Application → Bot：
   - `Reset Token` 获取 Token（只显示一次）
   - 开启 **MESSAGE CONTENT INTENT**（否则消息正文为空）
2. OAuth2 → URL Generator → Scopes 勾 `bot`，权限勾 `View Channels`、`Read Message History` → 打开 URL 授权进自己的服务器
3. 想转发**别人服务器**的频道（无法拉 Bot）：若对方是公告频道（有 Follow/关注 按钮），先关注到自己服务器的频道，再监听自己这个频道

## 常见问题

- **下拉列表是空的**：先填 Token 保存，等几秒让插件拉取完，再重新打开配置页；仍为空看日志（多为代理不通）。
- **HTTP 401**：Token 错误或已重置。
- **HTTP 403**：Bot 不在该服务器，或没有「查看频道 / 阅读消息历史」权限。
- **消息正文是空的**：没开 MESSAGE CONTENT INTENT。
- **收不到通知**：确认 设定→通知 渠道正常，且所选「通知类型」在目标渠道的消息类型开关里已打开。
- **微信里图片不显示**：企业微信服务器需能访问 Discord CDN，个别情况下图裂属正常，正文不受影响。
